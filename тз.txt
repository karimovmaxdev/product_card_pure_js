По апи приходит модель json (model.json) который содержит в себе информацию о товаре (скидках, размерах и так далее)

Структура у него следующая:

_id: айди модели в БД
name: 'Название модели'
product: Массив товаров которые относятся к этой модели
        [
            _id: айди товара в БД
            name: 'Название модели'
            artikul:Артикул товара
            guid: айди товара в 1с
            balance: Если товар не имеет размеров, то тут массив с размерамм (смотреть product.specifications.balance)
            price: Стоимость товара, так же если у товара нет размеров (смотреть product.specifications.price)
            sale:Процентная скидка
            status:Статус товара
            tnved: Тнвед товра
            color: Цвет товара
            ed_izm: Единица измерения
            activityspecifications: Содержит товар размеры или нет (генерируется на сервере, может не иметь значание)
            specifications:Размеры товара (если имеются)
                [
                    name: Hазвание размерам
                    guid: айди размера в 1с
                    artikul: Артикул размера
                    dimensions: Размеры габариты вес
                    barcode: Штрихкод товара
                    balance:Баланс товара по складам
                        [
                            name:Название склада
                            guid: айди склада в 1с
                            count: Количество остатка
                        ]
                    price:Стоимость размера
                           name:Название прайса
                           guid: айди прайса в 1с
                           count: стомость товара
                           currency: вид валюты
                ]
        ]
podzakaz: Продовать товар под заказ или нет
dimensions:Размеры габариты вес
marking: Товар имеет маркировку Честный знак


Задача:

1. Сверстать две страницы (для проверки основного функционала, красоты не требуется, должны быть элементы для проверки ниже описанного функционала).

   Первая страница каталог (название товара цена) - по клику переход на вторую страницу (catalog.html)
   Вторая страница товар (выбор цвета, название, цена, кнопки плюс-минус добавления товара) (product.html)

2. Основной функционал (JS):

   При переходе на страницу  (product.html) из model.json должен создаваться class Product на каждый выбранный цвет в зависимости от их смены, и непосредственно
   в html выводить данные с class
   (
    пример:
    class Product {
        ......
        _id:айди товара
        specifications:выбранный размер
        name:название товара
        и т.д (на усмотрения разработчика)
        ....

    } 
    let model = 'model.json'
    let pr = new Product(model.product)
   
   )

3. class Product должен уметь:

  - Останавливать увелечения количества товара для покупки при достижении баланса (если не стоит podzakaz)
  - При выборе размера, рассчитывать скидку если такая имеется
  - Если размер отсутствует в нужном количестве не давать его выбрать  (если не стоит podzakaz)
  - Если в самом продукты, отсутствуют габариты - установить их из самой модели
  - Менять выбранный размер (specifications) в самом классе выброного цвета
  - Менять стоимость в зависимости от выбранного размера
  - При смене размеров, пересчитывать данные которые есть в классе (стоимость, скидки,баланс)
  - Проверять баланс,остаток (2-Готовая продукция) только этот склад, на страницы данные только с него
  - Проверять статус размера, включен или нет. Если нет то не показывать его в выдаче размеров

Все данные должны рассчитываться автоматически, при замене model.json на другую модель
